# Install

LazyAgents ships two binaries — `lad` (the daemon) and `la` (the TUI). One install puts both on your `$PATH`.

> **Heads up — v1 platform scope.** Releases are produced for Linux x86_64/aarch64, macOS x86_64/aarch64, and Windows x86_64. Only Linux is currently validated end-to-end. The other targets are wired and built, but cross-platform smoke testing is a release-blocker for GA, not Beta. See [Troubleshooting → Platform notes](troubleshooting.md#platform-notes) for what to expect on macOS and Windows.

## Pick one

### Linux / macOS — `install.sh`

```sh
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/TinyWhite1997/LazyAgents/releases/latest/download/lazyagents-installer.sh | sh
```

The installer drops `la` and `lad` into `~/.cargo/bin` (or `~/.local/bin`, depending on what it finds first), and verifies the artifact attestation against sigstore.

### Windows — `install.ps1`

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://github.com/TinyWhite1997/LazyAgents/releases/latest/download/lazyagents-installer.ps1 | iex"
```

### Homebrew (status: **pending tap publication**)

A Homebrew formula is generated by every release but the `TinyWhite1997/homebrew-tap` repository is not yet published. Until then, use `install.sh`. The intended command will be:

```sh
brew install TinyWhite1997/homebrew-tap/lazyagents
```

### Scoop

Not currently shipped. Use `install.ps1` on Windows. A Scoop bucket is tracked as a follow-up item.

### From source

You need Rust **1.75** or newer (MSRV). The repo pins **1.96.0** for CI; any toolchain ≥ 1.75 will compile.

```sh
git clone https://github.com/TinyWhite1997/LazyAgents.git
cd LazyAgents
cargo install --path crates/la-daemon --locked
cargo install --path crates/la-tui    --locked
```

`la` and `lad` will land in `~/.cargo/bin`.

## Runtime prerequisites

| Requirement | Why |
|---|---|
| **A real terminal emulator.** Anything with ANSI / VT support — Kitty, Alacritty, WezTerm, iTerm2, Windows Terminal. | The TUI uses 256-colour and OSC sequences. |
| **`git` ≥ 2.20 on `$PATH`** (only if you use the worktree review feature). | `lad` shells out to `git worktree`, `git apply`, `git diff-tree`. Error parsing assumes English locale phrases from modern git. |
| **Windows 10 build 1809 or later** (Windows users only). | ConPTY is the only supported PTY backend on Windows. |

You do **not** need a system SQLite — LazyAgents bundles it. You do **not** need Node.js, Python, or any container runtime.

## Verify your install

```sh
la --version
lad --version
lad doctor
```

`lad doctor` prints the resolved socket and state directories, plus whether a daemon is already listening. Sample output:

```text
socket path:    /run/user/1000/lazyagents/lad-1.sock
runtime dir:    /run/user/1000/lazyagents
state dir:      /home/alice/.local/share/lazyagents
server version: 0.1.0
status:         no daemon listening
```

That last line is expected on a fresh install — `la` will start the daemon on first launch.

## Verify the release signature (optional)

LazyAgents binaries are signed via [GitHub Artifact Attestations](https://docs.github.com/en/actions/security-guides/using-artifact-attestations-to-establish-provenance-for-builds) (sigstore-backed, keyless). If you have `gh` installed:

```sh
gh attestation verify ./lazyagents-x86_64-unknown-linux-gnu.tar.xz \
  --repo TinyWhite1997/LazyAgents
```

## Where things live

| Thing | Path | Override |
|---|---|---|
| Config file | `$XDG_CONFIG_HOME/lazyagents/config.toml` (Linux) | `LAZYAGENTS_CONFIG_HOME` |
| SQLite database | `$XDG_DATA_HOME/lazyagents/lad.sqlite` | `LAZYAGENTS_DATA_DIR` |
| Session transcripts (when > 8 MiB each) | `$XDG_DATA_HOME/lazyagents/sessions/*.log` | (follows `LAZYAGENTS_DATA_DIR`) |
| Per-session git worktrees | `$XDG_DATA_HOME/lazyagents/worktrees/<project>/<sid>/` | (follows `LAZYAGENTS_DATA_DIR`) |
| Unix socket | `$XDG_RUNTIME_DIR/lazyagents/lad-1.sock` | `LAZYAGENTS_RUNTIME_DIR` |
| Windows pipe | `\\.\pipe\lazyagents-lad-1` | — |

Fallbacks: `$XDG_DATA_HOME` → `~/.local/share`; `$XDG_CONFIG_HOME` → `~/.config`; `$XDG_RUNTIME_DIR` → `/tmp/lazyagents-<UID>`.

## Update

`la --check-update` queries GitHub Releases and tells you whether a newer build exists. It never installs automatically — re-run the installer above to upgrade.

## Uninstall

```sh
rm -f ~/.cargo/bin/la ~/.cargo/bin/lad   # or ~/.local/bin/{la,lad}
rm -rf ~/.local/share/lazyagents         # SQLite + transcripts + worktrees (your data!)
rm -rf ~/.config/lazyagents              # UI preferences
```

LazyAgents does not write into your backends' data directories — uninstalling does not touch `~/.claude/projects/`, `~/.codex/sessions/`, or `~/.local/share/opencode/`. Those belong to those tools.
